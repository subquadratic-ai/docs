# Integration overview

Aldea is a standard REST and WebSocket API. You call it directly with any HTTP client without requiring any proprietary SDK. If you're migrating from Deepgram, you can also reuse Deepgram SDKs by pointing them at Aldea.

## Endpoints and protocols

The Aldea API provides two transcription modes, each with its own protocol and use case, as shown:

| | Pre-recorded | Streaming |
|---|---|---|
| **Protocol** | HTTPS (REST) | WebSocket |
| **Endpoint** | `POST https://api.aldea.ai/v1/listen` | `wss://api.aldea.ai/v1/listen` |
| **Auth** | `Authorization: Bearer org_...` | `Authorization: Bearer org_...` |
| **Input** | Audio bytes in request body | Binary audio frames over WebSocket |
| **Output** | Single JSON response with full transcript | Stream of JSON messages (interim + final) |
| **Use case** | Files, URLs, batch processing | Live audio, real-time captioning, voice assistants |

## Supported platforms

Aldea supports multiple platforms and works with any language that has an HTTP and WebSocket client.

| Platform | REST | WebSocket |
|----------|------|-----------|
| **Python** | `httpx` | `websockets` |
| **Node.js** | `fetch` (built-in) | `ws` |
| **Go** | `net/http` | `nhooyr.io/websocket` |
| **Rust** | `reqwest` | `tokio-tungstenite` |
| **Swift (iOS)** | `URLSession` | `URLSessionWebSocketTask` |
| **Kotlin (Android)** | OkHttp | OkHttp WebSocket |
